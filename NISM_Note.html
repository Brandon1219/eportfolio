UML (unified modelling language)

My note:

Action - use arrow with multiplicity mark

Inheritance: (IS-A) - use empty arrow
The children class can reuse the definition from the parent class.

Composition (HAS-A) - use diamond arrow
The child cannot be used without parent

Aggregation (USES-A) - use empty diamond arrow

liskov substitution principle

The Liskov Substitution Principle in practical software development. The principle defines that objects of a superclass shall be replaceable with objects of its subclasses without breaking the application. That requires the objects of your subclasses to behave in the same way.

data encapsulation:

A program development plan that involves a prototype using global variables and a final version that makes the global variables into instance attributes.
